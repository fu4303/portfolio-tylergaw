Papersaver

By Tyler Gaw

Paper is an excellent product. Like all great products, it not only does a thing well, it also makes you want to do that thing more. Paper makes me want to draw more. After drawing with Paper for a couple weeks, I wanted a quick way to share my drawings from Paper to the Web. I built Papersaver to allow me to do that.

In Dependence

Made with Paper is a cool showcase of what people are drawing with Paper. I went to submit one of my goofy characters to it and saw that they ask for an attribution link. They suggest–and most submissions are hosted on–Tumblr. Twitter and Facebook are also mentioned as hosting choices. I’m not a fan of that. I want the canonical location of my drawings to be on a server that I control. This is my natural tendency, but I was also inspired by Jeremy and Frank’s articles on the topic of pushing ourselves to take more control over our digital artifacts by self-hosting.

At this point it was clear that I had a fun side project in front of me. I had a few days off work for the holidays, so I started letting my brain run wild with possibilities. My plan was to build some type of site where I could publish my drawings and at the same time dig into technologies that were new to me.

Planning

I generally start a new project out with a few bullet points for what I want, what I don’t, technologies I’ll use, etc. These points don’t always make it into the final project and some are just questions to think about during the process. Here’s a few I jotted down for Papersaver:

Project Goals

  • Publish images from Paper to my personal Web site
  • Do not rely on third-party platforms for storage
  • “In Dependence”

What does it do?

  • Static HTML site
  • Displays a list of images (maybe thumbnails + click to enlarge)
  • Git repo backed up to Github
  • Different input methods: email, Twitter, direct to repo
  • generate(): image creation, HTML generation, Git commit, publish
  • How can others use this?
  • Permalinks to each item

Getting Rolling

After a few hours of writing down goals and researching similar projects I had a plan for what I would build and the tools I would use to build it. I wanted to be able to publish drawings directly from the Paper app. Paper allows drawings to be shared to Tumblr, Facebook, Twitter, Camera Roll, and Email. The first three were out since they’re third-parties. I entertained the idea of using a combination of export to cameral roll and Dropbox to publish, but went away from it because it didn’t feel as flexible as it should be. I’d be going the route of publishing by email.

Here’s the process for using Papersaver:

  • Draw in Paper
  • Email each drawing to a dedicated email address
  • A Node.js script would listen for new email at that address
  • When a new email arrives, that script would grab the attached image and hand it off to Papersaver.js
  • Papersaver.js will create the necessary directories and files that static site generator Wintersmith needs
  • Wintersmith would then regenerate the site with the new drawings
  • The newly added and created files would be commited to the Git repo and those commits would be pushed to the remote repo at Github
  • The drawings would then be available at http://lab.tylergaw.com/papers
